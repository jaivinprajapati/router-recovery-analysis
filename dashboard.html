Router Recovery — Live Dashboard

Wiom Analytics  |  Auto-refreshed daily  |  Logic aligned with Metabase dashboard #1067 (PICKUP Tickets)

1. Monthly Success Rate — creation month basis

i
Table: PROD_DB.DYNAMODB_READ.TASKS (deduped by ID, latest row)
Filter: TYPE = 'ROUTER_PICKUP'
21-day rule: CREATED (IST) between 2025-11-01 and (today − 21 days). Only tickets whose recovery window has fully expired are included.
Month: DATE_TRUNC('month', CREATED + 330 min IST)
Successfully Closed: STATUS = 2
Unsuccessfully Closed: STATUS = 3
Open: STATUS IN (0, 1)
Rate: Success ÷ Total Closed
Only tickets created ≥21 days ago (full recovery window expired). Prevents skew from fresh tickets still in progress.

2. Week-on-Week — 7-day rolling Success / Total Closed

i
Tables: PROD_DB.DYNAMODB.TASK_PERFORMANCE (tp) LEFT JOIN PROD_DB.DYNAMODB_READ.TASKS (deduped) ON tp.REPORTERID = TASKS.ID
Filter: tp.TASK_TYPE = 'ROUTER_PICKUP'  |  tp._FIVETRAN_DELETED = false  |  tp.TASK_RESOLVED_TIME NOT NULL
Date: tp.TASK_RESOLVED_TIME (+ 330 min IST), range 2025-11-01 → today
Successfully Closed: TASKS.STATUS = 2
Total Closed: all tp rows resolved that day
Rolling: 7-day trailing sum(Success) ÷ sum(Total Closed)
Smoothed daily rate. Resolution-date basis — no cohort lag, shows operational performance in near-real-time.

3. Day-on-Day — Success / Total Closed

i
Tables / filter / date: same as View 2 (TASK_PERFORMANCE ⟕ TASKS, resolution date)
Rate: (rows where TASKS.STATUS = 2) ÷ (all rows resolved that day) — no smoothing
Of all tickets closed that day, what % were successfully closed. Noisy but earliest signal.

4. Open Ticket Age Distribution

i
Table: PROD_DB.DYNAMODB_READ.TASKS (deduped by ID, latest row)
Filter: TYPE = 'ROUTER_PICKUP'  |  STATUS IN (0, 1)  |  CREATED (IST) ≥ 2025-11-01
Age: DATEDIFF(day, CREATED + 330 min IST, CURRENT_DATE)
Buckets: 0-7, 8-14, 15-21, 21+ days
Tickets still open. 21+ day bucket = SLA breach (partners have 21 days).
